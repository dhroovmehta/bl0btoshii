## Prompt Tips

- **If the eyes come out wrong** (white sclera with pupils), add: "simple closed curved line eyes, no eyeballs, no eye whites, chibi style eyes"
- **If the style is too chunky/NES**, add: "high detail, smooth shading, NOT 8-bit, NOT retro NES"
- **If the style is too smooth/not pixel art**, add: "clearly visible pixel grid, crisp pixel edges, no anti-aliasing on edges"
- **If the character doesn't match**, try generating 4 options and picking the closest, then iterate on the prompt
- **For consistency across characters**, always keep the shared style block at the end of every prompt

# BACKGROUND LAYER PROMPTS

**Target Style:** Bootoshi-style parallax backgrounds — layered pixel art with visible pixel grid, rich atmospheric colors, warm lighting, depth through color temperature (warm foreground, cool background). Think indie RPG overworld meets Studio Ghibli palette.

**Reference:** The Bootoshi.ai screenshot — green grass foreground, castle/buildings midground, soft clouds and sky background. Each layer has distinct depth and the overall scene feels like a cozy pixel art game world.

**Format:** Each location needs 3-4 separate PNG layers that stack on top of each other:
1. `background.png` — Sky, distant scenery (farthest, moves slowest)
2. `midground.png` — Buildings, furniture, structures (transparent background)
3. `foreground.png` — Ground, floor where characters stand (transparent above ground)
4. `effects.png` — Optional: light rays, steam, particles (transparent, overlays everything)

**Resolution:** Generate at 1920x1080 (16:9). The pipeline expects this exact size.

**IMPORTANT:** For layers 2, 3, and 4 — you MUST specify "transparent background" so they can be stacked.

---

## How To Prompt Backgrounds

Background prompts work differently than character prompts. You need to generate each layer separately:

1. Generate the **background layer** first (full scene, sky and distant elements)
2. Use that background as reference, then ask for the **midground layer** with transparent background
3. Then the **foreground layer** (just the ground/floor)
4. Optionally the **effects layer** (light, particles)

For each layer, upload the previous layer as reference so the style and color palette stay consistent.
